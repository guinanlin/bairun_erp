# 采购单保存接口（关联销售订单）- 前端调用说明

保存采购单时，若需要与销售订单双向关联（采购单上的「Order Confirmation No」+ 两侧 Connections 显示），请按本文传参。

---

## 1. 关联效果说明

| 位置 | 效果 |
|------|------|
| **采购单（Purchase Order）** | 主表「Order Confirmation No」显示销售订单号；打开采购单时 Connections 中可看到关联的 Sales Order。 |
| **销售订单（Sales Order）** | 打开销售订单时 Connections 中可看到关联的 Purchase Order。 |

实现方式：

- 主表传 **order_confirmation_no** = 销售订单 name → 采购单表单上显示订单确认号。
- 子表每行传 **sales_order** + **sales_order_item** → 采购单 Connections 显示 SO；保存后接口会回写 Sales Order Item 的 purchase_order，使销售订单 Connections 显示本采购单。

---

## 2. 接口基本信息

| 项目 | 说明 |
|------|------|
| **接口名称** | 保存采购单（单张） |
| **请求路径** | `POST /api/method/bairun_erp.utils.api.buying.purchase_order_add.save_purchase_order` |
| **认证** | 需登录（Cookie 或 `Authorization: Bearer {token}`） |
| **Content-Type** | `application/json` |

---

## 4. 与销售订单关联的字段说明

### 4.1 主表（采购单头）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| supplier | string | 是 | 供应商 |
| company | string | 是 | 公司 |
| order_confirmation_no | string | 否（关联时建议传） | **订单确认号**，填销售订单的 name（如 SAL-ORD-2026-00001），会在采购单上显示并用于业务关联。 |
| order_confirmation_date | string | 否 | 订单确认日期，格式 YYYY-MM-DD |
| transaction_date | string | 否 | 交易日期，默认当天 |
| schedule_date | string | 否 | 计划日期 |
| set_warehouse | string | 否 | 默认仓库 |
| 其他 | - | 否 | naming_series、taxes 等见后端 docstring |

### 4.2 子表（items 每一行）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| item_code | string | 是 | 物料编码 |
| qty | number | 建议 | 数量 |
| rate | number | 建议 | 单价 |
| sales_order | string | 关联时必填 | **销售订单 name**。与主表 order_confirmation_no 一致即可。不传则采购单 Connections 不会显示该 SO。 |
| sales_order_item | string | 关联时必填 | **Sales Order Item 的 name**（子表行唯一标识）。必填后接口才会回写 SO 行上的 purchase_order，销售订单的 Connections 才会显示本采购单。 |
| warehouse | string | 建议 | 仓库 |
| schedule_date | string | 建议 | 计划日期 |
| cost_center / expense_account | string | 视需求 | 成本中心 / 费用科目 |

**重要：**  
- 若只传主表 `order_confirmation_no` 而不传行的 `sales_order` / `sales_order_item`，采购单上能看到订单确认号，但 **Connections 不会建立**，销售订单侧也不会出现本采购单。  
- 要两侧 Connections 都生效，**每一行都要传 `sales_order` 和 `sales_order_item`**，且 `sales_order_item` 必须是该销售订单在 ERPNext 里对应行的 name（从销售订单接口或列表获取）。

---

## 5. 前端如何拿到 sales_order_item

- **从销售订单详情接口**：若你们有「根据销售订单 name 拉取明细」的接口，返回的每条明细里需包含该行的 **name**（即 Sales Order Item 的 name，通常为系统生成的短 id）。  
- **从 ERPNext 标准接口**：例如 `GET /api/resource/Sales Order/{so_name}` 或列表接口中展开 items，每条 item 的 `name` 即为 `sales_order_item`。  
- **创建采购单时**：若当前页是从某张销售订单进入的，列表/表单里应已有销售订单 name 和每行的 name，直接把这些值赋给 `order_confirmation_no`、`items[].sales_order`、`items[].sales_order_item` 即可。

---

## 6. 响应格式

**成功：**

```json
{
  "data": {
    "success": true,
    "name": "PUR-ORD-2026-00046",
    "message": "采购订单已保存并提交",
    "doc": { ... }
  }
}
```

**失败：**

```json
{
  "error": "错误信息"
}
```

---

## 7. 小结（给前端）

1. 要采购单上显示「Order Confirmation No」并和销售订单关联：主表传 **order_confirmation_no** = 销售订单号。  
2. 要采购单、销售订单两侧 Connections 都能看到对方：  
   - 主表传 **order_confirmation_no**；  
   - items 每一行传 **sales_order**（同订单号）和 **sales_order_item**（该行在 ERPNext 中的 name）。  
3. **sales_order_item** 必须从销售订单明细数据里取「行 name」，不能自己编。
