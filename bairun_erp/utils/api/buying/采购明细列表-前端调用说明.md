# 采购明细列表 - 前端调用说明

给前端工程师对接「采购明细」页用的接口说明，替换原来的「getDocList(Purchase Receipt) + 多次 getDoc」方式。

---

## 1. 接口基本信息

| 项目 | 说明 |
|------|------|
| **接口名称** | 采购明细列表（已入库收货单行扁平列表） |
| **请求路径** | `POST /api/method/bairun_erp.utils.api.buying.purchase_receipt_list.get_purchase_receipt_details_list` |
| **认证** | 需登录（Cookie 或 `Authorization: Bearer {token}`） |
| **Content-Type** | `application/json` |

---

## 2. 请求参数

**POST body 示例：**

```json
{
  "json_data": {
    "filters": [],
    "order_by": "posting_date desc, receipt_name asc, idx asc",
    "limit_start": 0,
    "limit_page_length": 50,
    "search_customer_order": "",
    "search_supplier": "",
    "search_item_name": ""
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| filters | array | 否 | 额外过滤条件，一般传 `[]` |
| order_by | string | 否 | 排序，默认 `posting_date desc, receipt_name asc, idx asc` |
| limit_start | number | 否 | 分页起始下标，从 0 开始 |
| limit_page_length | number | 否 | 每页条数；0 表示不分页（慎用） |
| search_customer_order | string | 否 | 销售订单号模糊筛选 |
| search_supplier | string | 否 | 供应商编号/名称模糊筛选 |
| search_item_name | string | 否 | 物料名称模糊筛选 |

| 顶层字段 | 类型 | 说明 |
|----------|------|------|
| message | array | 当前页的采购明细行数组 |

**每行字段（与列表列对应）：**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| receipt_name | string | 收货单号 |
| purchase_order | string | 采购订单号 |
| customer_order | string | 销售订单号 |
| supplier | string | 供应商编号 |
| supplier_name | string | 供应商名称 |
| posting_date | string | 单据日期 YYYY-MM-DD |
| item_code | string | 物料编码 |
| item_name | string | 物料名称 |
| order_qty | number | 订单数量 |
| received_qty | number | 入库数量 |
| outstanding_qty | number | 未到数量 |
| rate | number | 单价 |
| amount | number | 金额 |
| billed_amt | number | 已开票金额 |
| invoiced_percent | number | 开票进度 0–100 |
| warehouse | string | 仓库 |
| warehouse_slot | string | 库位 |
| rowKey | string | 行唯一键，用于列表 key |

**无数据时：** `message: []`，`total_count: 0`。

## 4. 前端对接要点

1. **替换旧逻辑**  
   采购明细页改为只调本接口，不再使用「getDocList(Purchase Receipt) + 对每条收货单 getDoc」的方式。

2. **分页与总数**  
   - 「共 X 条记录」使用响应里的 **total_count**。  
   - 分页请求时传 **limit_start**、**limit_page_length**（例如第 2 页每页 50 条：`limit_start: 50, limit_page_length: 50`）。

3. **筛选项**  
   销售订单号、供应商、物料名称三个筛选项对应传 **search_customer_order**、**search_supplier**、**search_item_name**（有输入时传字符串，无输入可传空字符串或不传）。

4. **表格列**  
   列顺序与字段对应关系保持现有设计即可：单据日期、销售订单号、物料名称、供应商编号、供应商名称、订单数量、入库数量、未到数量、单价、金额、开票进度、仓库、库位。

5. **行 key**  
   列表渲染时用 **rowKey** 作为行唯一 key。

## 6. 错误与未登录

- **未登录**：接口会返回 401/403，前端提示并展示空表或错误信息即可。  
- **参数错误**：可能返回 400 或 200 带错误信息，建议在 message 中展示原因。  
- **日期/空值**：日期为 YYYY-MM-DD；空值可能为 `null` 或省略，前端显示「—」即可。

如有字段名或枚举与现有前端类型不一致，以本说明为准做一次对齐即可。
