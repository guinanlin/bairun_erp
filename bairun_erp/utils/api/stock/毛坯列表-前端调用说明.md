# 毛坯列表 - 前端调用说明

给前端工程师对接「毛坯页」`/scm/mes/blank`（待委外 / 已委外两个 Tab）用的接口说明。

---

## 1. 接口概览

| 项目 | 说明 |
|------|------|
| **页面** | 毛坯页，路由 `/scm/mes/blank`，Tab：待委外 \| 已委外 |
| **接口数量** | 2 个：待委外列表、已委外列表 |
| **认证** | 需登录（Cookie 或 `Authorization: Bearer {token}`） |
| **Content-Type** | `application/json` |

---

## 2. 待委外列表

### 2.1 基本信息

| 项目 | 值 |
|------|-----|
| **接口名称** | 毛坯待委外列表 |
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.blank_list.get_pending_outsourcing_list` |
| **说明** | 返回「已入毛坯仓、且委外余数 > 0」的行，供「待委外」Tab 使用。行粒度：销售订单号 + 物料 + 仓库。 |

### 2.2 请求参数

**POST body 示例：**

```json
{
  "json_data": {
    "limit_start": 0,
    "limit_page_length": 50,
    "order_by": "project_no asc, item_code asc",
    "search_project_no": "",
    "search_outsourcing_supplier": ""
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| limit_start | number | 否 | 分页起始下标，从 0 开始，默认 0 |
| limit_page_length | number | 否 | 每页条数，默认 50；传 0 表示不分页（慎用） |
| order_by | string | 否 | 排序，默认 `project_no asc, item_code asc` |
| search_project_no | string | 否 | 销售订单号模糊筛选 |
| search_outsourcing_supplier | string | 否 | 委外供应商编码/名称模糊筛选 |
| filters | array | 否 | 预留，一般传 `[]` |
| status | string | 否 | 预留 |

### 2.3 响应格式

**成功时：** 响应体即 `{ "message": [...], "total_count": N }`，无再包一层 `data`。

```json
{
  "message": [
    {
      "id": "SO-2024-001|ITEM-001|毛坯 - B",
      "projectNo": "SO-2024-001",
      "itemCode": "ITEM-001",
      "itemFullName": "毛坯件A",
      "orderQty": 1000,
      "receivedQty": 500,
      "receiptDetails": [
        { "receiptDate": "2026-03-01", "receiptNo": "MAT-STE-2026-00001", "qty": 200 },
        { "receiptDate": "2026-03-05", "receiptNo": "MAT-STE-2026-00002", "qty": 300 }
      ],
      "unreceivedQty": 500,
      "warehouse": "毛坯 - B",
      "warehouseLocation": "",
      "outsourcingSupplierId": "",
      "outsourcingSupplier": "",
      "outsourcingQty": 0,
      "outsourcingDetails": [],
      "outsourcingRemainder": 500,
      "packingQty": null,
      "boxConfig": null,
      "volume": null,
      "unitPrice": null,
      "totalAmount": null,
      "workInstructionUrl": null,
      "status": "待委外"
    }
  ],
  "total_count": 42
}
```

| 顶层字段 | 类型 | 说明 |
|----------|------|------|
| message | array | 当前页的行数组 |
| total_count | number | 符合条件的**总条数**（用于「共 X 条记录」和分页） |

**无数据时：** `message: []`，`total_count: 0`。

## 3. 已委外列表

### 3.1 基本信息

| 项目 | 值 |
|------|-----|
| **接口名称** | 毛坯已委外列表 |
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.blank_list.get_outsourced_list` |
| **说明** | 返回「已有委外调拨（委外数 > 0）」的行，供「已委外」Tab 使用。行结构与待委外一致，仅数据源和 `status` 不同。 |

### 3.2 请求参数

与「待委外列表」相同，放在 `json_data` 中：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| limit_start | number | 否 | 分页起始下标 |
| limit_page_length | number | 否 | 每页条数 |
| order_by | string | 否 | 排序，默认 `project_no asc, item_code asc` |
| search_project_no | string | 否 | 销售订单号模糊筛选 |
| search_outsourcing_supplier | string | 否 | 委外供应商模糊筛选 |

### 3.3 响应格式

与待委外列表**同一套行结构**，区别仅在于：

- `status` 固定为 `"已委外"`。
- `outsourcingQty`、`outsourcingDetails` 有值（来自已提交的调拨单）；`outsourcingRemainder` = 到库数 − 委外数。

## 4. 行对象字段说明（两个接口共用）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | string | 行唯一键，建议作为列表 key、勾选、编辑标识 |
| projectNo | string | 销售订单号 |
| itemCode | string | 物料编码 |
| itemFullName | string | 物料名称（展示用） |
| orderQty | number | 订单数（该销售订单该物料的数量） |
| receivedQty | number | 到库数（已入毛坯仓数量） |
| receiptDetails | array | 到库明细，用于弹窗「到库明细」 |
| unreceivedQty | number | 未到库数（订单数 − 到库数） |
| warehouse | string | 仓库（一般为「毛坯 - B」） |
| warehouseLocation | string | 库位，暂无则空字符串 |
| outsourcingSupplierId | string | 委外供应商 ID/编码 |
| outsourcingSupplier | string | 委外供应商名称 |
| outsourcingQty | number | 委外数（**毛坯件数**） |
| outsourcingDetails | array | 委外明细，用于弹窗「委外明细」 |
| outsourcingRemainder | number | 委外余数（到库数 − 委外数，毛坯件） |
| packingQty | number \| null | 装箱数，当前多为 null，前端可显示「—」 |
| boxConfig | string \| null | 箱规，当前多为 null |
| volume | number \| null | 体积，当前多为 null |
| unitPrice | number \| null | 委外单价，当前多为 null |
| totalAmount | number \| null | 委外金额，当前多为 null |
| workInstructionUrl | string \| null | 作业指导书 URL，当前多为 null |
| status | string | `"待委外"` 或 `"已委外"` |

### 4.1 receiptDetails 每项

| 字段 | 类型 | 说明 |
|------|------|------|
| receiptDate | string | 入库日期 YYYY-MM-DD |
| receiptNo | string | 入库单号（Stock Entry 单号） |
| qty | number | 到库数量 |

### 4.2 outsourcingDetails 每项

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 委外日期 YYYY-MM-DD |
| docNo | string | 委外单号（调拨单号） |
| qty | number | 委外数量（毛坯件数） |

## 5. 前端对接要点

1. **两个 Tab 对应两个接口**  
   - 「待委外」Tab 调用：`get_pending_outsourcing_list`  
   - 「已委外」Tab 调用：`get_outsourced_list`

2. **分页与总数**  
   - 「共 X 条记录」使用响应里的 **total_count**。  
   - 翻页时传 **limit_start**、**limit_page_length**（例如第 2 页每页 50 条：`limit_start: 50, limit_page_length: 50`）。

3. **筛选项**  
   - 销售订单号 → **search_project_no**  
   - 委外供应商 → **search_outsourcing_supplier**  
   - 有输入时传字符串，无输入可传空字符串或不传。

4. **行 key**  
   - 列表渲染时用 **id** 作为行唯一 key。

5. **可选字段**  
   - packingQty、boxConfig、volume、unitPrice、totalAmount、workInstructionUrl 当前多为 `null`，前端可统一显示「—」。

## 6. 调用示例（fetch）

### 待委外列表

```javascript
const res = await fetch(
  '/api/method/bairun_erp.utils.api.stock.blank_list.get_pending_outsourcing_list',
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    credentials: 'include',
    body: JSON.stringify({
      json_data: {
        limit_start: 0,
        limit_page_length: 50,
        order_by: 'project_no asc, item_code asc',
        search_project_no: searchProjectNo || '',
        search_outsourcing_supplier: searchOutsourcingSupplier || '',
      },
    }),
  }
);
const data = await res.json();
const list = data.message || [];
const totalCount = data.total_count ?? 0;
```

## 7. 错误与未登录

- **未登录**：返回 401/403，前端提示并展示空表或错误信息。  
- **参数错误**：可能返回 400 或 200 带错误信息，建议在 message 中展示原因。  
- **日期/空值**：日期为 YYYY-MM-DD；空值可能为 `null` 或省略，前端显示「—」即可。

## 8. 后端字段排查与数据来源说明

若列表中**大部分字段显示为空或 0**，可按下表核对：接口返回的每行对象**字段名均为驼峰**，数值为 **number** 类型（非字符串），字符串无值时为 `""`。

### 8.1 行对象字段与数据来源（与前端排查表一一对应）

| 序号 | 字段名（驼峰） | 类型 | 后端数据来源 | 何时会空/为 0 |
|------|----------------|------|--------------|----------------|
| 1 | id | string | `{projectNo}\|{itemCode}\|{warehouse}` 拼接 | 不会空 |
| 2 | projectNo | string | 入库单(SE) → reference_purchase_receipt → PR 的 `customer_order` 或 PR 子表同物料的 `sales_order` | **PR 无该字段或 PR 行未填销售订单时为 `""`** |
| 3 | itemCode | string | 入库/调拨的物料编码 | 不会空（有行必有物料） |
| 4 | itemFullName | string | `Item` 主数据的 `item_name` | 物料不存在或无名称时为 `""` |
| 5 | orderQty | number | `Sales Order Item` 按 (销售订单 name, item_code) 汇总的 stock_qty/qty | **当 projectNo 为空时无法关联 SO，故为 0** |
| 6 | receivedQty | number | 入毛坯仓的 Stock Entry 行 qty 按 (projectNo, itemCode, warehouse) 汇总 | 有入库则有值 |
| 7 | receiptDetails | array | 同上，每条入库 SE 一行：`{ receiptDate, receiptNo, qty }` | 无入库明细则为 `[]` |
| 8 | unreceivedQty | number | orderQty − receivedQty（不小于 0） | 随 orderQty/receivedQty 变化 |
| 9 | warehouse | string | 固定为「毛坯 - B」或从数据带出 | 不会空 |
| 10 | warehouseLocation | string | 当前未接库位数据 | 固定 `""` |
| 11 | outsourcingSupplierId | string | 调拨单(SE) 的 `custom_outsourcing_supplier` | 待委外或无该字段时为 `""` |
| 12 | outsourcingSupplier | string | Supplier 的 supplier_name | 同上 |
| 13 | outsourcingQty | number | 毛坯→半成品调拨按同维度汇总的 qty（毛坯件数） | 待委外多为 0；已委外有调拨则有值 |
| 14 | outsourcingDetails | array | 每条调拨一行：`{ date, docNo, qty }` | 无委外明细则为 `[]` |
| 15 | outsourcingRemainder | number | receivedQty − outsourcingQty（不小于 0） | 有到库则有值 |
| 16–21 | packingQty, boxConfig, volume, unitPrice, totalAmount, workInstructionUrl | number/string null | 当前未接 Item/BOM 等 | 固定为 `null`，前端显示「—」即可 |
| 22 | status | string | 待委外接口固定 `"待委外"`，已委外接口固定 `"已委外"` | 不会空 |

### 8.2 为何 projectNo / orderQty 常为空或 0？

- **projectNo**：来自「入库单(SE) → 关联的 PR → 销售订单」。若 PR 主表没有 `customer_order`、且 PR 子表该物料行没有填 `sales_order`，则 projectNo 为 `""`，列表「销售订单号」列会空。
- **orderQty**：来自 Sales Order Item，按 **(销售订单 name, 物料)** 汇总。只有 projectNo 有值（即能对应到某张 Sales Order）时，才会查到该订单下该物料的订单数；**projectNo 为空时 orderQty 恒为 0**。

**建议**：在采购接收(PR)或 PR 行上维护与销售订单的关联（如 bairun_erp 的 `customer_order` 或子表 `sales_order`），并确保入库链（PR → 质检 → SE 入毛坯仓）中 PR 带有该关联，待委外列表即可带出销售订单号与订单数。
