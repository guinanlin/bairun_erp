# 生产仓库（成品 / 原材料 / 库存仓）- 前端调用说明

给前端工程师对接「成品」「原材料」「库存仓」三个页面用的接口说明。  
**接口模块**：`bairun_erp.utils.api.stock.inventory`。

---

## 1. 接口概览

| 项目 | 说明 |
|------|------|
| **页面** | 成品 `/scm/mes/finished`、原材料 `/scm/mes/raw-materials`、库存仓 `/scm/mes/inventory` |
| **接口数量** | 13 个：3 个列表 + 10 个写操作 |
| **认证** | 需登录（Cookie 或 `Authorization: Bearer {token}`） |
| **Content-Type** | `application/json` |
| **基础路径** | `POST /api/method/bairun_erp.utils.api.stock.inventory.<方法名>` |

### 1.1 接口清单

| 仓库 | 方法名 | 说明 |
|------|--------|------|
| 成品 | `get_finished_list` | 列表（已入库 / 已出库） |
| 成品 | `outbound_finished` | 出库 |
| 成品 | `transfer_finished_to_inventory` | 移库（成品仓 → 库存仓） |
| 原材料 | `get_raw_material_list` | 列表（待入库 / 已入库 / 已出库） |
| 原材料 | `inbound_raw_material` | 入库 |
| 原材料 | `outbound_raw_material` | 出库 |
| 库存仓 | `get_inventory_list` | 列表（待入库 / 已入库 / 已出库） |
| 库存仓 | `get_inventory_reservation_details` | 预留明细（按行查） |
| 库存仓 | `submit_inventory_to_warehouse` | 仓库提交 |
| 库存仓 | `inbound_inventory` | 统计入库 |
| 库存仓 | `outbound_inventory` | 出库 |
| 库存仓 | `other_inbound_inventory` | 其他入库 |
| 库存仓 | `other_outbound_inventory` | 其他出库 |

### 1.2 状态枚举（传参用）

| 前端 Tab | 传参 `status` |
|----------|----------------|
| 待入库 | `pending_inbound` |
| 已入库 | `in_stock` |
| 已出库 | `outbound` |

成品只有「已入库」「已出库」，对应 `in_stock`、`outbound`。

## 2. 成品仓（/scm/mes/finished）

### 2.1 成品列表 `get_finished_list`

| 项目 | 值 |
|------|-----|
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.inventory.get_finished_list` |
| **说明** | 按状态返回「已入库」或「已出库」行；支持销售订单号筛选、分页。 |

**请求 body（推荐放在 `json_data` 里）：**

```json
{
  "json_data": {
    "status": "in_stock",
    "search_project_no": "",
    "limit_start": 0,
    "limit_page_length": 50,
    "order_by": "project_no asc, item_code asc"
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | `in_stock` 已入库（默认）、`outbound` 已出库 |
| search_project_no | string | 否 | 销售订单号模糊筛选 |
| limit_start | number | 否 | 分页起始下标，默认 0 |
| limit_page_length | number | 否 | 每页条数，默认 50 |
| order_by | string | 否 | 排序 |

**响应：** `{ "message": [ 行对象, ... ], "total_count": N }`

**行对象字段（成品）：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 行唯一键，格式 `projectNo\|itemCode\|warehouse`，用于勾选、出库/移库传参 |
| date | string | 单据日期 YYYY-MM-DD |
| projectNo | string | 销售订单号 |
| itemFullName | string | 存货全名 |
| orderQty | number | 订单数 |
| receivedQty | number | 到库数 |
| unreceivedQty | number | 未到库数 |
| warehouse | string | 仓库（如 成品 - B） |
| warehouseLocation | string | 库位 |
| packingQty | number \| null | 装箱数 |
| boxConfig | string \| null | 箱规 |
| volume | number \| null | 体积 |
| unitPrice | number | 单价（成本单价） |
| workInstructionUrl | string | 作业指导书 URL |
| status | string | `"已入库"` 或 `"已出库"` |

---

### 2.2 成品出库 `outbound_finished`

| 项目 | 值 |
|------|-----|
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.inventory.outbound_finished` |
| **说明** | 勾选「已入库」行的 id 列表，扣减成品仓并生成出库单。 |

**请求 body：**

```json
{
  "json_data": {
    "ids": ["projectNo|itemCode|成品 - B", "..."]
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ids | string[] 或 string | 是 | 行 id 数组；或逗号分隔的 id 字符串 |

**响应：**  
成功：`{ "success": true, "stock_entry": "MAT-STE-xxx" }`  
失败：`{ "success": false, "error": "错误信息" }`

---

### 2.3 成品移库 `transfer_finished_to_inventory`

| 项目 | 值 |
|------|-----|
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.inventory.transfer_finished_to_inventory` |
| **说明** | 成品仓 → 库存仓（或指定目标仓），生成调拨单。 |

**请求 body：**

```json
{
  "json_data": {
    "ids": ["projectNo|itemCode|成品 - B", "..."],
    "target_warehouse": "库存仓 - B"
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ids | string[] 或 string | 是 | 行 id 列表 |
| target_warehouse | string | 否 | 目标仓库，默认 `"库存仓 - B"` |

**响应：** 同出库，成功带 `stock_entry`，失败带 `error`。

## 4. 库存仓（/scm/mes/inventory）

### 4.1 库存仓列表 `get_inventory_list`

| 项目 | 值 |
|------|-----|
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.inventory.get_inventory_list` |
| **说明** | 按状态返回待入库/已入库/已出库；支持采购订单号筛选；可选带预留明细。 |

**请求 body：**

```json
{
  "json_data": {
    "status": "in_stock",
    "search_project_no": "",
    "limit_start": 0,
    "limit_page_length": 50,
    "include_reservation_details": false
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | `pending_inbound` / `in_stock` / `outbound` |
| search_project_no | string | 否 | 采购订单号模糊 |
| limit_start / limit_page_length | number | 否 | 分页 |
| include_reservation_details | boolean | 否 | 为 true 时每行带 `reservationDetails` 数组 |

**响应：** `{ "message": [ 行对象, ... ], "total_count": N }`

**行对象字段（库存仓）：** 在原材料基础上增加：

| 字段 | 类型 | 说明 |
|------|------|------|
| reservedQty | number | 预留数 |
| reservationDetails | array | 仅当请求中 `include_reservation_details: true` 时存在，见下 |

**reservationDetails 每项：**

| 字段 | 类型 | 说明 |
|------|------|------|
| salesContract | string | 销售合同号 |
| productName | string | 产品名称 |
| reservedQty | number | 预留数量 |
| customer | string | 客户 |

---

### 4.2 预留明细 `get_inventory_reservation_details`

| 项目 | 值 |
|------|-----|
| **请求地址** | `POST /api/method/bairun_erp.utils.api.stock.inventory.get_inventory_reservation_details` |
| **说明** | 按行 id 或 item_code+warehouse 查该行预留明细（弹窗只读）。 |

**请求 body（二选一）：**


```json

```

**响应：** `{ "reservationDetails": [ { "salesContract", "productName", "reservedQty", "customer" }, ... ] }`

### 4.6 其他入库 `other_inbound_inventory`

| 项目 | 值 |

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|

| qty（或 数量） | number | 是 | 数量，> 0 |
| reason / source / ref_no / warehouse_location / remarks | string | 否 | 写入单据备注等 |

## 5. 前端对接要点

1. **列表与 Tab**  
   - 成品：「已入库」Tab 传 `status: "in_stock"`，「已出库」传 `status: "outbound"`。  
   - 原材料 / 库存仓：三个 Tab 分别传 `pending_inbound`、`in_stock`、`outbound`。

2. **分页与总数**  
   - 使用响应里的 **total_count** 做「共 X 条」和分页；翻页传 **limit_start**、**limit_page_length**。

3. **筛选**  
   - 成品：**search_project_no** = 销售订单号。  
   - 原材料 / 库存仓：**search_project_no** = 采购单号 / 采购订单号。

4. **行 key 与写操作**  
   - 列表行用 **id** 作为 key；出库、移库、入库、仓库提交等传 **ids** 数组（或逗号分隔字符串），元素即为列表行的 **id**。

5. **写操作响应**  
   - 成功：`success: true`，多数带 `stock_entry`（单据号）。  
   - 失败：`success: false`，`error` 为错误文案，前端可 toast/message 展示。

## 7. 错误与未登录

- **未登录**：返回 401/403，前端提示并展示空表或错误信息。  
- **无权限**：写操作需「Stock Entry」创建权限，列表需「Warehouse」读权限；无权限时接口抛错，前端可统一捕获并提示。  
- **业务失败**：写操作返回 `{ "success": false, "error": "..." }`，前端用 `error` 文案提示。  
- **空值**：日期为 YYYY-MM-DD；空值可能为 `null` 或 `""`，前端显示「—」即可。
