# 生产仓库（成品 / 原材料 / 库存仓）— 需求分析与后端对接说明

**文档版本**：2026-03  
**目标读者**：后端工程师  
**关联需求**：《生产仓库（成品 / 原材料 / 库存仓）后端对接需求文档》  
**前端状态**：成品、原材料、库存三个页面已实现，部分为 Mock；需后端按本文档理解需求并实现/补齐接口。  
**后端实现路径**：白名单方法统一放在 `bairun_erp.utils.api.stock.inventory`（即 `apps/bairun_erp/bairun_erp/utils/api/stock/inventory.py`）。

---

## 1. 需求理解摘要

### 1.1 业务范围

- **「生产仓库」** 在前端是侧栏的一个**菜单分组**，其下包含：毛坯、半成品、**成品、原材料、库存** 等子菜单；每个子菜单对应**一个具体仓库**的库存/单据明细页。
- **本文档仅覆盖其中三个仓库**，且前端已有对应页面、交互模式相近，便于统一对接：
  - **成品仓**：如 ERP 中的「成品 - B」
  - **原材料仓**：如「原材料仓 - B」
  - **库存仓**：如「库存仓 - B」

### 1.2 状态模型（与前端 Tab 一致）

| 仓库     | 前端路径                    | Tab 状态                         | 说明 |
|----------|-----------------------------|----------------------------------|------|
| 成品     | `/scm/mes/finished`         | **已入库**、**已出库**           | 无「待入库」 |
| 原材料   | `/scm/mes/raw-materials`    | **待入库**、**已入库**、**已出库** | 与采购单关联 |
| 库存仓   | `/scm/mes/inventory`        | **待入库**、**已入库**、**已出库** | 与采购单关联，且需**预留明细**子表 |

后端需能按「**仓库类型 + 状态**」查询明细，并支持各页面上的写操作（入库、出库、移库、其他入库/出库等）。

### 1.3 与现有实现的对照

| 维度         | 毛坯/半成品（已实现）              | 成品/原材料/库存仓（本文档）     |
|--------------|------------------------------------|----------------------------------|
| 接口模块     | `bairun_erp.utils.api.stock.blank_list` | `bairun_erp.utils.api.stock.inventory`（待实现） |
| 列表形态     | 按 `list_type`（blank/semi_finished）+ 状态（待委外/已委外） | 按**仓库类型**（finished / raw_material / inventory）+ **状态**（待入库/已入库/已出库） |
| 原材料列表   | —                                  | 已有 `bairun_erp.utils.api.material.item.get_raw_material_item`，需保证字段一致并可接入「按状态」列表 |
| 数据源       | SE/PR/销售订单 + 毛坯仓/半成品仓   | 各仓库的库存/采购单/销售预留等  |

## 2. 需求分析（按仓库）

### 2.1 成品仓（/scm/mes/finished）

**理解要点**

- 成品无「待入库」Tab，只有**已入库**、**已出库**，即只关心「已在成品仓」与「已从成品仓发出」两类。
- 行维度与**销售订单**强绑定：筛选用「销售订单号」，字段含 `projectNo`（销售订单号）、`orderQty`/`receivedQty`/`unreceivedQty`、`packingQty`、`boxConfig`、`volume`、`unitPrice` 等。
- **操作**：刷新、添加一行（已入库）、**移库**（成品库 → 库存仓库）、**出库**（已入库 → 已出库）。

**后端需要**

1. **列表接口**  
   - 按仓库类型 = 成品（如 `成品 - B`）、状态 = 已入库 / 已出库 查询；支持按销售订单号筛选；分页可选。  
   - 返回字段与需求文档 §2.2 对齐（id、date、projectNo、itemFullName、orderQty、receivedQty、unreceivedQty、warehouse、warehouseLocation、packingQty、boxConfig、volume、unitPrice、workInstructionUrl、status 等）。
2. **出库接口**  
   - 入参：勾选行的 id 或唯一键。  
   - 逻辑：扣减成品库库存，并将这些行/单据状态更新为已出库。
3. **移库接口**  
   - 入参：勾选行 + 目标仓库（库存仓）。  
   - 逻辑：成品库 → 库存仓的调拨（生成 Stock Entry 或等价调拨单），库存仓侧可后续「接收」确认。

**数据源思路（供实现参考）**

- 已入库：成品仓（`成品 - B`）的库存或入库明细，按销售订单 + 物料 + 仓库维度聚合；状态=已入库。
- 已出库：从成品仓发出的出库/调拨记录，按相同维度聚合；状态=已出库。
- 与现有 `blank_list` 类似，可按 Warehouse + Stock Ledger / Stock Entry 等组合查询，需与 ERP 实际 Doctype/字段对齐。

---

### 2.2 原材料仓（/scm/mes/raw-materials）

**理解要点**

- 三个 Tab：**待入库**、**已入库**、**已出库**；与**采购单**关联（筛选为采购单号）。
- 前端已对接 `get_raw_material_item` 拉列表，但该接口当前更偏「物料主数据 + 在库数」，未必完全覆盖「按采购单 + 状态」的列表；若需「待入库/已入库/已出库」分 Tab，后端需提供**按状态过滤的列表**或扩展现有接口。
- **操作**：刷新、添加原材料物料、**入库**（待入库→已入库）、**出库**（已入库→已出库）；分页。

**后端需要**

1. **列表接口**  
   - 支持参数：`page`、`page_length`（或 `page_size`）、`status`（待入库/已入库/已出库）、可选采购单号筛选。  
   - 返回字段与需求文档 §3.2 一致（id、date、projectNo、itemFullName、orderQty、unitPrice、receivedQty、unreceivedQty、inStockQty、supplierId、supplier、inventoryCost、salesPrice、warehouse、warehouseLocation、unit、workInstructionUrl、status）。
2. **入库接口**  
   - 勾选「待入库」行确认入库：更新状态为已入库并写入库存（如生成/更新 PR 或 Stock Entry)。
3. **出库接口**  
   - 勾选「已入库」行出库：扣减库存并更新状态为已出库。
4. **添加物料**  
   - 若前端新增一行需落库，可提供单条创建/保存接口；否则可与现有物料主数据接口复用。

**与现有 `get_raw_material_item` 的关系**

- 保持现有接口的返回字段结构（如 `itemFullName`、`inStockQty`、`warehouse` 等），便于前端复用。
- 「按状态列表」可新写在 `inventory` 模块（如 `get_raw_material_list`），按仓库类型=原材料 + status 查询；或与 material 模块协商由一方统一提供，但**白名单入口**按需求应落在 `inventory.py` 中供前端统一调用。

---

### 2.3 库存仓（/scm/mes/inventory）

**理解要点**

- 同样为**待入库、已入库、已出库**三 Tab；与**采购订单**关联；比原材料多「**预留明细**」：每行可有 `reservationDetails` 数组（销售合同号、产品名称、预留数量、客户等）。
- **操作**：刷新、**仓库提交**、**统计入库**、**出库**、**其他入库**、**其他出库**；预留明细为弹窗只读。

**后端需要**

1. **列表接口**  
   - 按仓库类型 = 库存仓（如 `库存仓 - B`）、状态 = 待入库/已入库/已出库 查询；支持按采购订单号筛选。  
   - 返回主表字段（与需求文档 §4.2 一致）；`reservationDetails` 可本接口一并返回，或单独按行 id 查询预留明细接口。
2. **仓库提交**  
   - 勾选「待入库」行提交给仓库；业务语义由后端定义（如状态不变仅标记「已提交」，或驱动工作流）。
3. **统计入库**  
   - 勾选「待入库」行：状态改为已入库并写入库存。
4. **出库**  
   - 勾选「已入库」行：扣减库存并更新为已出库。
5. **其他入库**  
   - 弹窗字段：业务原因、来源、单据号、物料、数量、库位、备注；后端生成库存仓入库记录。
6. **其他出库**  
   - 弹窗字段：业务原因、去向、单据号、物料、数量、备注；后端扣减库存仓并可选生成出库明细。

**数据源思路**

- 主表：库存仓的采购订单行/库存明细，按状态区分待入库/已入库/已出库。
- 预留明细：销售合同或销售订单对库存的预留关系表，按存货+仓库汇总得到 `reservationDetails`。

## 3. 统一对接要点（后端实现约定）

### 3.1 列表接口形态

- **统一形态**：按 **仓库类型 + 状态** 查询明细；支持订单号类筛选与分页。
- **仓库类型** 与 ERP 的 name/code 对应建议：
  - 成品：`finished` / `成品 - B`
  - 原材料：`raw_material` / `原材料仓 - B`
  - 库存仓：`inventory` / `库存仓 - B`
- **状态枚举** 建议：`pending_inbound`（待入库）、`in_stock`（已入库）、`outbound`（已出库）；成品仅后两种。

### 3.2 写操作与权限

- 入库、出库、移库、仓库提交、其他入库/出库等，均需后端提供明确接口，并在接口层做权限校验（如 `frappe.has_permission` 或自定义角色）。
- 行内编辑：若需即时落库可提供 PATCH 单行接口；否则在「提交」「入库」等动作时批量提交变更。

### 3.3 主数据与下拉

- 仓库、库位、供应商、存货、销售/采购订单号等下拉数据，可与现有主数据或 ERP 接口复用；缺失时由后端单独提供列表接口。

### 3.4 白名单方法存放路径

- **路径**：`bairun_erp.utils.api.stock.inventory`（即 `apps/bairun_erp/bairun_erp/utils/api/stock/inventory.py`）。
- 成品、原材料、库存仓三块与「生产仓库」相关的**白名单方法**（如 `get_finished_list`、`get_raw_material_list`、`get_inventory_list`、`outbound_finished`、`transfer_finished_to_inventory`、`inbound_raw_material`、`outbound_raw_material`、`inbound_inventory`、`outbound_inventory`、`other_inbound_inventory`、`other_outbound_inventory` 等）**统一写在该模块**，便于前端统一调用与权限配置。
- 与现有 `blank_list.py`（毛坯/半成品）并列，同属 `bairun_erp.utils.api.stock` 包下。

## 4. 接口清单汇总（供实现核对）

| 仓库     | 接口类型   | 建议方法名（示例）           | 说明 |
|----------|------------|------------------------------|------|
| 成品     | 列表       | `get_finished_list`          | 按状态 已入库/已出库，销售订单号筛选，分页 |
| 成品     | 出库       | `outbound_finished`          | 勾选已入库行出库 |
| 成品     | 移库       | `transfer_finished_to_inventory` | 成品库 → 库存仓 |
| 原材料   | 列表       | `get_raw_material_list`      | 按状态 待入库/已入库/已出库，采购单号筛选，分页；可与现有 get_raw_material_item 字段对齐 |
| 原材料   | 入库       | `inbound_raw_material`       | 待入库 → 已入库 |
| 原材料   | 出库       | `outbound_raw_material`      | 已入库 → 已出库 |
| 原材料   | 添加物料   | 可选，或复用 material 模块  | 新增一行落库 |
| 库存仓   | 列表       | `get_inventory_list`        | 按状态 + 采购订单号筛选，可选带 reservationDetails |
| 库存仓   | 预留明细   | `get_inventory_reservation_details` | 按行 id 返回预留明细（若列表不直接带） |
| 库存仓   | 仓库提交   | `submit_inventory_to_warehouse` | 待入库行提交给仓库 |
| 库存仓   | 统计入库   | `inbound_inventory`         | 待入库 → 已入库 |
| 库存仓   | 出库       | `outbound_inventory`        | 已入库 → 已出库 |
| 库存仓   | 其他入库   | `other_inbound_inventory`   | 弹窗字段落库 |
| 库存仓   | 其他出库   | `other_outbound_inventory`  | 弹窗字段扣减并出库 |

以上方法名仅为示例，实现时可按团队命名习惯调整；**实现位置**以 `inventory.py` 为准。
