# 采购价格「成本明细按工艺更新」— 前端对接说明

## 文档信息

| 项目 | 说明 |
|------|------|
| 业务说明 | 按 **物料编码 + 工艺** 更新 `Item` 子表 **成本明细**（`BR Item Cost Detail`）中的**一行** |
| 请求方式 | `POST` |
| 鉴权 | 需登录态；当前用户需具备 **物料（Item）写权限** |
| 接口路径 | `/api/method/bairun_erp.utils.api.buying.purchase_price.update_item_cost_detail_by_process` |

---

## 1. 与列表接口的关系

- 列表/查询仍使用：`get_material_details_by_category`（见《采购价格-按工艺获取物料明细-前端对接说明》）。
- 列表返回的每一行若来自成本明细子表，会带有 **`process`**（对应子表字段 `br_process`）。更新本接口时，`process` 参数应与之对应。
- 子表行的唯一标识为 **`cost_detail_name`**（即 Frappe 子表行的 `name`，形如随机字符串）。当同一物料下**存在多条相同工艺**时，更新接口**必须**传入 `cost_detail_name`，否则会报错。

---

## 2. 请求参数

推荐使用 `json_data` 包裹参数（与项目内其他采购接口一致）：

```json
{
  "json_data": {
    "item_code": "STO-ITEM-2026-00062",
    "process": "注塑",
    "cost_detail_name": "",
    "updates": {
      "br_daily_output": 3600,
      "br_unit_product_cost": 0.2143,
      "br_audit_status": "已审核",
      "br_auditor": "张三"
    }
  }
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `item_code` | string | 是 | 物料编码，对应 `Item.name` |
| `process` | string | 是 | 工艺；与 `br_process` 二选一（见下） |
| `br_process` | string | 否 | 与 `process` 同义 |
| `cost_detail_name` | string | 否 | 子表行 `name`；**同一物料多条同工艺时必填** |
| `row_name` | string | 否 | 与 `cost_detail_name` 同义 |
| `updates` | object | 条件 | 要更新的字段键值对；**至少传一个可更新字段** |
| `fields` | object | 否 | 与 `updates` 同义，二者可合并 |

也可将「可更新字段」直接放在 `json_data` **顶层**（与 `updates` 合并），例如：

```json
{
  "json_data": {
    "item_code": "STO-ITEM-2026-00062",
    "process": "注塑",
    "br_daily_output": 3600,
    "br_audit_status": "已审核"
  }
}
```

### 2.1 工艺白名单（`process` / `br_process`）

仅支持以下值（与列表接口 `category` 白名单一致）：

`注塑, UV镀, 罩光, 喷涂, 水镀, 滴油, 点钻, 组装, 手工活, 饰品配件, 烫金, 印刷, 玻璃瓶, 水转印, 热转印, 植绒`

---

## 3. 可更新字段白名单（`updates` 的 key）

以下字段名**必须与后端一致**（为 `BR Item Cost Detail` 上的字段名）。**未列出的字段**（含只读/派生字段）不可传入，否则返回 `不允许更新的字段: ...`。

| 字段名 | 界面含义 | 说明 |
|--------|----------|------|
| `br_injection_molding_per_day` | 注塑/天 | Float，非负 |
| `br_cavity_count` | 1出几 | Int，非负 |
| `br_cycle_time` | 周期 | Float，非负；单位一般为秒 |
| `br_raw_material` | 原材料 | 文本 |
| `br_price_per_gram` | 克单价 | Currency |
| `br_weight_grams` | 重量 (克) | Float |
| `br_daily_output` | 每天出产量 | Float |
| `br_unit_product_cost` | 单个产品成本 | Currency |
| `br_auditor` | 审核人 | 文本 |
| `br_audit_status` | 审核状态 | `已审核` 或 `未审核` |

**不要传入**（保存时由后端子表逻辑自动计算或固定）：

- `br_material_cost_yuan`（材料成本 = 克单价 × 重量）
- `br_seconds_per_hour`（固定为 3600）

**不要通过本接口修改** `br_process`（工艺）：定位行时以入参 `process` 为准；若需改工艺，应在 ERP 物料表单中操作或另行扩展接口。

---

## 4. 行定位规则（重要）

1. 在 `item_code` 对应物料的 **`br_cost_details`** 中查找 **`br_process` 等于请求 `process`** 的行。
2. **0 行**：报错，提示未找到该工艺的成本明细行。
3. **1 行**：更新该行。
4. **多行**：报错，提示须传入 `cost_detail_name`。此时应用 **`get_material_details_by_category` 或其它途径拿到的子表行 `name`**（若列表接口未直接返回 `cost_detail_name`，需从物料表单 API / `frappe.client.get` 等获取子表行的 `name` 字段）。

若直接传入 `cost_detail_name`：

- 按子表 `name` 精确定位一行；
- 且该行的 **`br_process` 必须与 `process` 一致**，否则报错。

---

## 5. 响应格式

成功时 HTTP 200，典型结构：

```json
{
  "message": {
    "ok": true,
    "item_code": "STO-ITEM-2026-00062",
    "cost_detail_name": "abc123xyz",
    "process": "注塑",
    "br_injection_molding_per_day": 0.011,
    "br_cavity_count": 1,
    "br_cycle_time": 23,
    "br_raw_material": "神华1102K PP原料",
    "br_price_per_gram": 0.1067,
    "br_weight_grams": 1.85,
    "br_material_cost_yuan": 0.1974,
    "br_seconds_per_hour": 3600,
    "br_daily_output": 3600,
    "br_unit_product_cost": 0.2143,
    "br_auditor": "张三",
    "br_audit_status": "已审核"
  }
}
```

前端一般读取 **`data.message`** 作为更新后的整行快照（含派生后的 `br_material_cost_yuan` 等）。

失败时 Frappe 返回 `exc` / `_server_messages` 等，常见文案包括：

| 场景 | 说明 |
|------|------|
| `item_code is required` / `process is required` | 缺参 |
| `invalid process: ...` | 工艺不在白名单 |
| `没有可更新的字段...` | `updates` 为空且顶层也无白名单字段 |
| `不允许更新的字段: ...` | 传了非白名单字段 |
| `没有修改物料的权限` | 无 Item 写权限 |
| `未找到该物料下工艺为「xxx」的成本明细行` | 无匹配行 |
| `该物料存在多条相同工艺的成本明细...` | 须传 `cost_detail_name` |
| `未找到 cost_detail_name 对应的成本明细行` | `name` 不存在 |
| `cost_detail_name 对应的行工艺与 process 不一致` | 工艺与行不匹配 |

## 6. 前端调用示例（TypeScript）

```ts
const res = await fetch(
  "/api/method/bairun_erp.utils.api.buying.purchase_price.update_item_cost_detail_by_process",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    credentials: "include",
    body: JSON.stringify({
      json_data: {
        item_code: "STO-ITEM-2026-00062",
        process: "注塑",
        // cost_detail_name: "xxx", // 多条同工艺时必填
        updates: {
          br_price_per_gram: 0.1067,
          br_weight_grams: 1.85,
          br_audit_status: "已审核",
          br_auditor: "张三"
        }
      }
    })
  }
);

const data = await res.json();
if (data.exc) {
  // 处理 Frappe 异常
  throw new Error(data.exc);
}
const row = data.message as {
  ok: boolean;
  item_code: string;
  cost_detail_name: string;
  process: string;
  br_material_cost_yuan: number;
  // ...
};
```

---

## 7. 联调注意事项

- 更新后 **`br_material_cost_yuan`** 会随克单价、重量重算，前端应以响应为准刷新展示。
- 若需展示/缓存 **子表行 `name`**，请保存 `cost_detail_name`，便于下次在「多行同工艺」场景下调用。
- 列表接口若需直接带 `cost_detail_name`，需评估是否要在 `get_material_details_by_category` 的返回中增加字段；当前更新接口已支持独立使用。
