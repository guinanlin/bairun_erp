## 一、业务概念梳理

### 1.1 页面与 Tab 含义

| 概念 | 含义 |
|------|------|
| **毛坯页** | MES 模块下「毛坯仓库委外」业务的列表页，管理已入库毛坯仓的毛坯件之委外计划与执行。 |
| **待委外** | 毛坯件已在毛坯仓库、尚未出库委外。列表按「**销售订单号 + 物料 + 仓库**」维度展示（一行一个组合），支持录入委外数、拆分行、打印+委外等。 |
| **已委外** | 毛坯件已从毛坯仓出库并委外。列表展示委外单号、委外日期、委外数等，用于查询与跟踪。 |

**委外业务在本系统中的定义**：不采用 ERPNext Subcontracting，也不自建「委外发料」DocType。从仓库视角，**委外 = 一次调拨**：毛坯仓（毛坯 - B）→ 半成品仓（半成品 - B）。即毛坯件从毛坯仓发出、经加工后以半成品形式入半成品仓；系统上体现为从毛坯仓转出、目标为半成品仓的 Stock Entry（调拨单）。数量上可按 BOM 换算（如 2 毛坯 → 1 半成品，则 100 件毛坯委外对应半成品仓入库 50 件）。

### 1.2 核心指标口径

| 指标 | 口径 |
|------|------|
| **到库数** | **基于销售订单**（非采购订单）：该**销售订单**下该物料，已入库到**毛坯仓**的数量合计。毛坯仓入库来源：采购接收（PR）→ 质检 → Stock Entry 入毛坯仓；PR 与 SE 上均有关联销售订单的字段，可按该字段汇总。 |
| **未到库数** | 订单数 − 到库数（尚未到毛坯仓的数量）。 |
| **委外数** | 按**毛坯件数**计。该行已执行（已提交调拨）的委外数量；待委外 Tab 可编辑的为前端草稿，未提交前不落库，列表展示的委外数/委外明细仅来自已提交的调拨单。 |
| **委外余数** | **到库数 − 委外数**（按毛坯件计）。例如到库 100 件、委外 80 件，则委外余数 = 20 件。 |

### 1.3 业务规则（与前端一致）

- 同一销售订单号下，所有行的「委外数」之和 ≤ 该订单该物料的「到库数」。
- 未到库数 = 订单数 − 到库数。
- 委外金额 = 委外数 × 委外单价（或由后端直接给出）。

## 二、数据源与列表逻辑

### 2.1 待委外列表

- **行粒度**：一行 = **销售订单号 + 物料 + 仓库**（不按库位再拆分，仅此三维度）。
- **数据源**：毛坯仓库存或毛坯仓入库明细，按上述三维度聚合。毛坯仓在 site2 中为固定仓库名 **毛坯 - B**（`tabWarehouse.name`），查询时按该仓库过滤即可；多站点/多公司若需兼容，可与业务确认是否用配置表维护毛坯仓名称。
- **筛选条件**：仅展示「仍在毛坯仓、尚未调拨至半成品仓」的毛坯（即未产生从毛坯 - B 到 半成品 - B 的调拨记录或未达委外数量）。
- **待委外时的委外数**：已录入但未提交的委外数**不在后端持久化**，仅前端本地/草稿保存；待委外列表接口返回的委外数、委外明细**仅来自已提交的调拨单**（Stock Entry）。

### 2.2 已委外列表

- **数据源**：委外在系统中体现为**毛坯仓 → 半成品仓**的调拨。即：从**毛坯 - B** 转出、目标为**半成品 - B** 的 **Stock Entry**（类型为 Material Transfer 或与业务约定的「毛坯委外」等）。
- **粒度**：每行对应一次调拨（或 Stock Entry 行），含单号、日期、物料、数量等。**委外供应商**：在调拨单（Stock Entry）上**增加委外供应商字段**（如 Link to Supplier 或自定义），用于列表展示与筛选。
- **说明**：不采用 ERPNext Subcontracting，也不自建委外发料 DocType；委外单证即上述调拨单（Stock Entry）。

### 2.3 与 bairun_erp / ERPNext 的对应关系（参考）

| 本需求含义 | 可能的数据来源（需按实际 Doctype/字段调整） |
|------------|---------------------------------------------|
| 销售订单号 | Sales Order.name；或库存/入库单上的 customer_order、project 等 |
| 订单数 | Sales Order Item.qty 或汇总 |
| 到库数 | **基于销售订单**（非采购订单）。**毛坯仓（毛坯 - B）** Stock Ledger 或 Material Receipt / Stock Entry 中，**该销售订单 + 该物料**的已入库数量汇总。 |
| 到库明细 | 入库到毛坯仓的 Material Receipt / Stock Entry 行：posting_date, name, qty |
| 仓库/库位 | 毛坯仓即 Warehouse「毛坯 - B」，半成品仓即「半成品 - B」；库位为库存维度或入库单上的 warehouse_slot |
| 委外供应商 | **在 Stock Entry（调拨单）上增加委外供应商字段**（如 Link to Supplier），用于展示与筛选；数据来自该调拨单上的供应商。 |
| 委外数/委外明细 | **毛坯仓→半成品仓**的 Stock Entry（调拨）行：posting_date, name, qty；**委外数按毛坯件数**计。委外明细即该类调拨单按销售订单+物料汇总得到的 date, docNo, qty 列表。 |
| 委外余数 | **到库数 − 委外数**（按毛坯件计），已确定。 |
| 箱规/体积/单价 | Item 或 BOM/工艺主数据、或单据行字段 |

**待委外 / 已委外区分**：以「是否有从毛坯仓（毛坯 - B）到半成品仓（半成品 - B）的调拨记录」为判定；待委外为在毛坯仓且未调拨（或未达已录委外数），已委外为已有对应调拨记录的汇总/列表。

## 三、接口约定梳理

### 3.1 通用约定

- **认证**：需登录态（Cookie 或 `Authorization: Bearer {token}`）。
- **Content-Type**：`application/json`。
- **日期格式**：YYYY-MM-DD；空值可为 `null` 或省略。
- **与现有 bairun_erp 列表接口一致**：建议请求使用 `json_data` 包裹参数；响应使用 `message`（行数组）+ `total_count`（总条数）。

### 3.2 待委外列表接口

| 项目 | 值 |
|------|-----|
| 接口名称 | 毛坯待委外列表 |
| 建议路径 | `POST /api/method/bairun_erp.utils.api.stock.blank_list.get_pending_outsourcing_list` |
| 说明 | 返回「毛坯已入库、尚未委外出库」的列表行，供「待委外」Tab 使用。 |

**请求参数（建议放入 json_data）**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| status | string | 否 | 固定 "pending" 或由后端约定 |
| filters | array | 否 | 额外过滤条件 |
| order_by | string | 否 | 排序，默认如 project_no asc, item_code asc |
| limit_start | number | 否 | 分页起始下标，从 0 开始 |
| limit_page_length | number | 否 | 每页条数；0 表示不分页（慎用） |
| search_project_no | string | 否 | 销售订单号（精确或模糊由后端约定） |
| search_outsourcing_supplier | string | 否 | 委外供应商编码/名称模糊 |

**响应**：`{ "message": [ 行对象, ... ], "total_count": 总条数 }`。无数据时 `message: []`，`total_count: 0`。

### 3.3 已委外列表接口

| 项目 | 值 |
|------|-----|
| 接口名称 | 毛坯已委外列表 |
| 建议路径 | `POST /api/method/bairun_erp.utils.api.stock.blank_list.get_outsourced_list` |
| 说明 | 返回「毛坯已出库委外」的列表行，供「已委外」Tab 使用。 |

**请求参数**：与待委外类似，建议增加 `status: "outsourced"` 或等价参数；排序可按委外日期倒序。

**响应**：与待委外同一套行结构，`status` 为 `"已委外"`；若业务上已委外行不需要到库明细/委外明细，可省略或精简。

### 3.4 可选：统一列表接口

- 一个接口同时支持两个 Tab：请求增加 `list_type`: `"pending"` | `"outsourced"`（或与 `status` 取值一致）。
- 响应结构不变，仅数据源与 `status` 按 `list_type` 不同而不同。

### 3.5 可选：下拉选项接口

为前端「销售订单号」「委外供应商」「仓库」「库位」筛选项及表格内选择提供数据：

- 销售订单号列表：当前有毛坯待委外/已委外数据的销售订单号集合（或全量，由业务决定）。
- 委外供应商列表：委外供应商编码 + 名称。
- 仓库列表：毛坯仓等可选的仓库。
- 库位列表：可按仓库过滤的库位列表。

接口路径与参数可与现有项目「下拉/选项」类 API 统一。

## 四、前端所需行结构（统一梳理）

以下为前端列表与弹窗所需字段，两个 Tab 共用同一套列结构，仅数据源与是否可编辑不同。

### 4.1 行对象字段

| 字段名 | 类型 | 说明 | 备注 |
|--------|------|------|------|
| id | string | 行唯一键 | 前端 key、勾选、编辑 |
| projectNo | string | 销售订单号 | 列表列、筛选 |
| itemFullName | string | 物料名称（展示） | 列表列 |
| itemCode | string | 物料编码 | 内部、跳转或接口关联 |
| orderQty | number | 订单数（该销售订单该物料的数量） | 列表列 |
| receivedQty | number | 到库数（已入毛坯仓数量） | 列表列 |
| receiptDetails | array | 到库明细（多次入库） | 弹窗「到库明细」 |
| unreceivedQty | number | 未到库数（订单数 − 到库数） | 列表列，建议后端计算 |
| warehouse | string | 仓库 | 列表列 |
| warehouseLocation | string | 库位 | 列表列 |
| outsourcingSupplierId | string | 委外供应商 ID/编码 | 内部、筛选 |
| outsourcingSupplier | string | 委外供应商名称 | 列表列、筛选 |
| outsourcingQty | number | 委外数（**毛坯件数**；列表与接口统一按毛坯件） | 列表列 |
| outsourcingDetails | array | 委外明细（多次委外） | 弹窗「委外明细」 |

| packingQty | number | 装箱数 | 列表列 |
| boxConfig | string | 箱规（如 20件/箱） | 列表列 |
| volume | number | 体积 | 列表列 |
| unitPrice | number | 委外单价 | 列表列 |
| totalAmount | number | 委外金额 | 列表列 |

| status | "待委外" \| "已委外" | 状态 | Tab 切换、是否可编辑 |

### 4.2 到库明细（receiptDetails）每项

| 字段 | 类型 | 说明 |
|------|------|------|
| receiptDate | string | 入库日期 YYYY-MM-DD |
| receiptNo | string | 入库单号 |
| qty | number | 到库数量 |

### 4.3 委外明细（outsourcingDetails）每项

委外在本系统中为「毛坯仓→半成品仓」调拨，故委外明细即对应 Stock Entry（调拨单）行。

| 字段 | 类型 | 说明 |
|------|------|------|
| date | string | 委外日期 YYYY-MM-DD（即调拨单 posting_date） |
| docNo | string | 委外单号（即 Stock Entry.name，调拨单号） |
| qty | number | 委外数量（**毛坯件数**，即该调拨行的 qty） |

## 五、与 bairun_erp 现有实现的对照

### 5.1 现有 API 风格

- **采购明细列表**（`purchase_receipt_list.get_purchase_receipt_details_list`）：POST，参数放在 `json_data`，返回 `message` + `total_count`；支持 `limit_start`、`limit_page_length`、`search_customer_order`、`search_supplier`、`search_item_name`。
- **采购订单列表**（`purchase_order_list`）：同样使用 `json_data`、`message`、`total_count`，以及 `search_customer_order`、`search_supplier` 等。

毛坯列表接口建议与上述风格一致：`json_data` 内传 `search_project_no`、`search_outsourcing_supplier`，以及分页、排序参数。

### 5.2 已有数据与术语

- **销售订单号**：bairun_erp 中多为 `customer_order`（主表自定义）或子表 `sales_order`；需求中的「销售订单号」对应前端 `projectNo`，后端可统一从 Sales Order.name 或 customer_order 取值，并在文档中注明映射。
- **仓库**：已有 `Warehouse`、`set_warehouse`、`warehouse`、`warehouse_slot`。site2 中**毛坯仓**为固定仓库名 **毛坯 - B**，**半成品仓**为 **半成品 - B**（均为 `tabWarehouse` 叶子仓）；质检入库等处曾出现「毛胚 - B」为笔误，应与「毛坯 - B」统一。
- **入库/库存与销售订单关联**：毛坯仓入库链路为 采购接收（PR）→ 质检 → Stock Entry 入毛坯仓；**PR 与 SE 上均有关联销售订单的字段**，到库数/到库明细可按该字段按「销售订单 + 物料」汇总或关联。

### 5.3 已确定与待确认部分

| 项目 | 说明 |
|------|------|
| **毛坯仓标识** | **已确定**。site2 中毛坯仓为固定仓库名 **毛坯 - B**（`tabWarehouse.name`），半成品仓为 **半成品 - B**；实现时按固定仓库名过滤即可。多站点/多公司若需兼容，可与业务确认是否用配置表维护。 |
| **委外单证** | **已确定**。不采用 ERPNext Subcontracting，也不自建委外发料 DocType。委外即「毛坯仓→半成品仓」的调拨，委外单证为 **Stock Entry**（Material Transfer 或与业务约定的类型），关键字段：单号、日期、物料、数量、来源仓库=毛坯 - B、目标仓库=半成品 - B；委外明细从该类 Stock Entry 按销售订单+物料汇总。**委外供应商**：在 Stock Entry 上**增加委外供应商字段**（如 Link to Supplier），用于展示与筛选。 |
| **委外余数** | **已确定**。委外余数 = **到库数 − 委外数**（按毛坯件计）。 |
| 箱规/体积/作业指导书 | 来自 Item、BOM 或工艺主数据，需确认来源字段与权限。 |

## 六、错误处理与对接说明

- **未登录**：返回 401/403，前端提示并展示空表或错误信息。
- **参数错误**：建议 400 或 200 + 业务错误码，message 中说明原因。
- **无数据**：返回 200，`message: []`，`total_count: 0`。
- **字段缺失**：可选字段可省略或为 `null`，前端做默认展示（如「—」）。
- **保存草稿、打印+委外（提交委外）** 等写操作接口可在本期或后续迭代单独约定；本梳理仅覆盖列表查询与展示所需的数据与接口。

## 八、接口实现说明（已实现）

- **实际 API 路径**  
  - 待委外列表：`POST /api/method/bairun_erp.utils.api.stock.blank_list.get_pending_outsourcing_list`  
  - 已委外列表：`POST /api/method/bairun_erp.utils.api.stock.blank_list.get_outsourced_list`  
  - 实现位置：`bairun_erp/utils/api/stock/blank_list.py`

- **Stock Entry 自定义字段（调拨单）**  
  - `custom_customer_order`：Link to Sales Order，对应前端 projectNo；毛坯委外调拨时必填，用于按销售订单汇总与筛选。  
  - `custom_outsourcing_supplier`：Link to Supplier，委外供应商；用于已委外列表展示与 search_outsourcing_supplier 筛选。  
  - 定义文件：`bairun_erp/custom/stock_entry.json`（需执行 migrate 或同步 Custom Field 后生效）。

- **销售订单号来源（projectNo）**  
  - **待委外列表**：到库数来自「入毛坯仓」的 Stock Entry（Material Receipt），通过 `Stock Entry Detail.reference_purchase_receipt` 关联 Purchase Receipt；projectNo = PR 主表 `customer_order`（若有）或 PR 子表同物料行的 `sales_order`。  
  - **已委外列表**：来自调拨单主表 `Stock Entry.custom_customer_order`（创建调拨单时需写入，否则该调拨无法按销售订单汇总）。

- **委外供应商来源**  
  - 来自调拨单主表 `Stock Entry.custom_outsourcing_supplier`；列表行上的 outsourcingSupplierId / outsourcingSupplier 取自该维度下调拨单的该字段及 Supplier.supplier_name。

- **字段映射（后端 → 前端）**  
  - 后端行对象字段名已与文档 §4.1 一致（projectNo、itemCode、receivedQty、outsourcingQty、outsourcingRemainder 等）；可选字段 packingQty、boxConfig、volume、unitPrice、totalAmount、workInstructionUrl 当前返回 null，前端可按「—」占位展示。
