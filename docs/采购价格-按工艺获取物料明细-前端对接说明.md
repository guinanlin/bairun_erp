# 采购价格「按工艺获取物料明细」— 前端对接说明

## 文档信息

| 项目 | 说明 |
|------|------|
| 适用页面 | `/scm/po/price-list` |
| 接口名称 | 按采购类目查询物料明细 |
| 请求方式 | `POST` |
| 鉴权 | 需登录态 |
| 接口路径 | `/api/method/bairun_erp.utils.api.buying.purchase_price.get_material_details_by_category` |

---

## 1. 请求参数

推荐使用 `json_data` 包裹参数（与现有项目接口风格一致）：

```json
{
  "json_data": {
    "category": "注塑",
    "table_form": "default",
    "limit_start": 0,
    "limit_page_length": 50,
    "order_by": "item_code asc",
    "search_item": ""
  }
}
```

参数定义：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `category` | string | 是 | 采购类目/工艺 |
| `process` | string | 否 | `category` 同义；二选一即可 |
| `table_form` | string | 否 | `default/form2/form3/form4`；不传则后端按类目自动映射 |
| `limit_start` | number | 否 | 分页起始，默认 `0` |
| `limit_page_length` | number | 否 | 每页条数，默认 `50`，最大 `500` |
| `order_by` | string | 否 | 排序，默认 `item_code asc` |
| `search_item` | string | 否 | 按物料编码/名称模糊搜索 |

### 1.1 类目白名单（`category`）

仅支持以下值：

`注塑, UV镀, 罩光, 喷涂, 水镀, 滴油, 点钻, 组装, 手工活, 饰品配件, 烫金, 印刷, 玻璃瓶, 水转印, 热转印, 植绒`

### 1.2 排序白名单（`order_by`）

仅支持以下字段（方向支持 `asc/desc`）：

- `item_code`
- `item_name`
- `modified`
- `modified_by`
- `creation`

示例：`item_name desc`

---

## 2. 响应格式

接口返回：

- `message`: 行数组
- `total_count`: 总条数（用于分页）

```json
{
  "message": [
    {
      "row_no": 1,
      "item_code": "24-043",
      "item_name": "ABS料本色外盖毛胚",
      "process": "注塑",
      "machine_cost_per_piece": 0.011,
      "output_per_shot": 1,
      "cycle_seconds": 23,
      "raw_material": "神华1102K PP原料",
      "price_per_gram": 0.1067,
      "weight_grams": 1.85,
      "material_cost_yuan": 0.1974,
      "cycle_per_hour": 3600,
      "daily_output": 3600,
      "estimated_cost_per_piece": 0.2143,
      "last_modifier": "Administrator",
      "last_modify_time": "2026-03-31 12:00:00",
      "auditor": "张三",
      "audit_status": "已审核",
      "form2_suppliers": [],
      "form4_suppliers": [],
      "category": "注塑",
      "table_form": "default"
    }
  ],
  "total_count": 128
}
```

---

## 3. 字段说明（行对象）

| 字段 | 类型 | 说明 |
|------|------|------|
| `row_no` | number | 返回结果序号（按当前分页结果连续） |
| `item_code` | string | 物料编码 |
| `item_name` | string | 物料名称 |
| `process` | string | 成本明细行所选工艺（子表 `br_process`） |
| `machine_cost_per_piece` | number/null | 注塑/天 |
| `output_per_shot` | number/null | 1出几 |
| `cycle_seconds` | number/null | 周期 |
| `raw_material` | string | 原材料 |
| `price_per_gram` | number/null | 克单价 |
| `weight_grams` | number/null | 重量(克) |
| `material_cost_yuan` | number/null | 材料成本(元) |
| `cycle_per_hour` | number/null | 周期1小时 |
| `daily_output` | number/null | 每天出产量 |
| `estimated_cost_per_piece` | number/null | 单个产品成本 |
| `last_modifier` | string | 最近修改人 |
| `last_modify_time` | string | 最近修改时间 |
| `auditor` | string | 审核人（来自成本明细子表） |
| `audit_status` | string | 审核状态（`已审核/未审核`） |
| `form2_suppliers` | array | 形式二/三供应商数组 |
| `form4_suppliers` | array | 形式四供应商数组 |
| `category` | string | 当前查询类目 |
| `table_form` | string | 当前行对应的表格形态 |

---

## 4. 多条成本明细的返回规则（重要）

同一物料如存在多条 `BR Item Cost Detail`，接口会**全部展开返回**：

- 一个物料可对应多条 `message` 行；
- 每条行数据对应一条成本明细记录；
- `total_count` 为展开后的总条数（不是物料去重数）。

---

## 5. 四种表格形态与供应商结构

### 5.1 `default`

- 主要使用标准成本字段；
- `form2_suppliers` / `form4_suppliers` 通常为空数组。

### 5.2 `form2`

用于：`UV镀/罩光/喷涂/水镀/水转印/热转印/植绒`

`form2_suppliers` 结构：

```json
[
  { "process": "喷涂", "work_station": "一工位", "unit_price": 0.12 },
  { "process": "喷涂", "work_station": "一工位", "unit_price": 0.14 }
]
```

### 5.3 `form3`

用于：`滴油/点钻/组装/手工活/饰品配件/烫金/印刷`

- 仍使用 `form2_suppliers`；
- `work_station` 为空字符串。

### 5.4 `form4`

用于：`玻璃瓶`

`form4_suppliers` 结构：

```json
[
  { "process": "玻璃瓶", "weight_grams": 25.3, "unit_price": 0.3, "amount": 7.59 }
]
```

## 6. 前端调用示例

```ts
const res = await fetch(
  "/api/method/bairun_erp.utils.api.buying.purchase_price.get_material_details_by_category",
  {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    credentials: "include",
    body: JSON.stringify({
      json_data: {
        category: "注塑",
        table_form: "default",
        limit_start: 0,
        limit_page_length: 50,
        order_by: "item_code asc",
        search_item: ""
      }
    })
  }
);

const data = await res.json();
const rows = data.message || [];
const total = data.total_count || 0;
```

---

## 7. 联调注意事项

- 若报 `invalid category`，请检查类目值是否在白名单内；
- 建议前端分页始终使用后端返回的 `total_count`；
- 数值字段可能为 `null`，前端做显示与计算时需兜底；
- 若新增了子表字段但接口未反映，先确认站点已执行 `bench --site <site> migrate`。
