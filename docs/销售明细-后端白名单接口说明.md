# 销售明细 - 后端白名单接口说明

## 一、接口概览

| 项目 | 说明 |
|------|------|
| **方法名** | `get_sales_order_details_list` |
| **完整路径** | `bairun_erp.utils.api.sales.sales_order_details_list.get_sales_order_details_list` |
| **ERPNext 调用 URL** | `POST /api/method/bairun_erp.utils.api.sales.sales_order_details_list.get_sales_order_details_list` |
| **认证** | 需登录（`allow_guest=False`） |
| **用途** | 销售明细页列表数据源，返回分页后的销售订单明细及 total，供前端映射为 ListContract |

通过 FastAPI 代理时，前端应调用现有 `POST /erpnext/resource`，将 `endpoint` 设为上述方法路径，`method` 为 POST，`json_data` 为下表参数。

---

## 二、请求参数（JSON body）

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `limit_start` | int | 否 | 0 | 分页偏移（从 0 开始） |
| `limit_page_length` | int | 否 | 20 | 每页条数 |
| `order_by` | str | 否 | `creation desc` | 排序，仅支持：`creation` / `modified` / `transaction_date` / `delivery_date` / `grand_total` / `name`，以及 `asc` / `desc` |
| `customer_name_search` | str | 否 | - | 客户名称或客户编码模糊搜索 |
| `order_date_from` | str | 否 | - | 下单日期起，格式 YYYY-MM-DD |
| `order_date_to` | str | 否 | - | 下单日期止，格式 YYYY-MM-DD |

**示例请求体：**

```json
{
  "limit_start": 0,
  "limit_page_length": 20,
  "order_by": "creation desc",
  "customer_name_search": "某某客户",
  "order_date_from": "2025-01-01",
  "order_date_to": "2025-12-31"
}
```

---

## 三、响应格式

- **结构**：`{ "data": [ {...}, ... ], "total": N }`
- **data**：当前页的销售订单明细列表，每项为一条 Sales Order 的聚合数据。
- **total**：符合筛选条件的总条数（用于分页展示「共 N 条」）。

### 单条记录字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| name | str | 销售订单编号 |
| transaction_date | str | 下单/交易日期 |
| creation | str | 创建时间 |
| customer | str | 客户编码 |
| customer_name | str | 客户名称 |
| po_no | str | 客户 PO 号 |
| currency | str | 币种 |
| grand_total | float | 订单总金额 |
| total_qty | float | 订单总数量 |
| docstatus | int | 单据状态（0 Draft, 1 Submitted, 2 Cancelled） |
| status | str | 业务状态 |
| owner | str | 创建人 |
| delivery_date | str | 交货日期 |
| per_billed | float | 已开票比例 |
| per_delivered | float | 已交货比例 |
| item_names | str | 子表物料名称拼接（顿号分隔） |
| delivered_qty | float | 子表已交数量汇总 |
| outstanding_qty | float | 未交数量（total_qty - delivered_qty） |
| （可选）billing_status | str | 开票状态（若 DocType 存在该字段） |
| （可选）custom_sub_order_type | str | 子订单类型（若存在自定义字段） |
| （可选）custom_contract_no / custom_style_number / custom_sales_qty / custom_marketing_fee | - | 自定义字段，若存在则返回 |

**示例响应：**

```json
{
  "data": [
    {
      "name": "SAL-ORD-2025-00001",
      "transaction_date": "2025-03-01",
      "creation": "2025-03-01 10:00:00.000000",
      "customer": "CUST-001",
      "customer_name": "某某客户",
      "po_no": "PO-2025-001",
      "currency": "CNY",
      "grand_total": 10000.0,
      "total_qty": 100.0,
      "docstatus": 1,
      "status": "To Deliver",
      "owner": "administrator@example.com",
      "delivery_date": "2025-03-15",
      "per_billed": 0,
      "per_delivered": 0,
      "item_names": "物料A、物料B",
      "delivered_qty": 0.0,
      "outstanding_qty": 100.0
    }
  ],
  "total": 1
}
```

## 四、调用方式

### 1. 在 ERPNext 内（bench execute）

```bash
bench --site site2.local execute bairun_erp.utils.api.sales.sales_order_details_list.get_sales_order_details_list --kwargs '{"limit_start": 0, "limit_page_length": 20}'
```

### 2. 通过 HTTP（需携带 Cookie / API Key / Token）

```http
POST /api/method/bairun_erp.utils.api.sales.sales_order_details_list.get_sales_order_details_list
Content-Type: application/json

{"limit_start": 0, "limit_page_length": 20, "order_by": "creation desc"}
```

### 3. 通过 FastAPI 代理（前端推荐）

前端调用现有 `getErpnextResource`（或等价代理）：

- **endpoint**：`/api/method/bairun_erp.utils.api.sales.sales_order_details_list.get_sales_order_details_list`
- **method**：POST
- **body.json_data**：上表参数（limit_start、limit_page_length、order_by、customer_name_search、order_date_from、order_date_to）

代理将请求转发至 ERPNext，并将返回的 `{ "data": [...], "total": N }` 原样返回给前端。

---

## 五、与销售合同 / 销售未交的区分

| 列表 | 数据源 | 说明 |
|------|--------|------|
| 销售合同列表 | 可继续用 GET /api/resource/Sales Order 或后续白名单 | 全部订单列表 |
| 销售未交列表 | 已有 per_delivered < 100 或专用白名单方法 | 未交数量 > 0 |
| **销售明细列表** | **本接口** `get_sales_order_details_list` | 全部订单的明细视图，含 item_names、delivered_qty、outstanding_qty |
