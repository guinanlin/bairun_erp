# 半成品 MES 委外（半成品→成品）— 后端白名单编排接口

## 一、接口概览

| 项目 | 说明 |
|------|------|
| **方法名** | `submit_semi_finished_outsourcing` |
| **完整路径** | `bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_semi_finished_outsourcing` |
| **ERPNext URL** | `POST /api/method/bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_semi_finished_outsourcing` |
| **认证** | 需登录（`allow_guest=False`），HTTP 方法 **POST** |
| **用途** | 与毛坯委外接口 **同一套编排**：MR（Material Issue，提交）→ SE（Material Issue，提交）→ SE（Material Receipt，**草稿**回库）→ PO（可选）。差异见下文「服务端默认」。 |

与毛坯入口对照文档：[毛坯委外-后端白名单编排接口说明.md](./毛坯委外-后端白名单编排接口说明.md)。

---

## 二、与 `submit_blank_outsourcing` 的差异（服务端默认）

| 维度 | 本接口行为 |
|------|------------|
| **MR 默认目标仓** | 未传 `target_warehouse` / `to_warehouse` 时，若存在与公司一致的 **`成品 - B`**（[`blank_list.FINISHED_WAREHOUSE`](../utils/api/stock/blank_list.py)），自动作为 MR 行目标仓；否则须显式传入与发料仓不同的有效仓库。 |
| **`custom_business_type`** | 未传或空字符串时，默认 **`半成品转成品`**（仍须与 Material Request 自定义字段可选值一致，否则提交 MR 可能失败）。 |
| **PO 服务行物料** | 未 `skip_purchase_order` 且未传 `po_service_item_code` / `po_service_item_name` 时，默认按 Item **`item_name` =「半成品委外」** 解析服务物料（与毛坯侧默认「毛坯委外」对称）。主数据须存在对应 Item，或通过上述参数显式指定。 |

其余业务规则（回库推导、SO+BOM、Stock Entry 定制字段、PO 合并等）与毛坯文档 **第二节** 描述一致：发料 `item_code` 在 BOM 中的**直接父项**作为回库物料；数量按销售订单与多级 BOM 展开计算。

---

## 三、请求参数（`json_data`）

字段表与毛坯接口相同，见 [毛坯委外-后端白名单编排接口说明.md 第三节](./毛坯委外-后端白名单编排接口说明.md)。本入口典型用法：

- `from_warehouse`：半成品源仓（如 **`半成品 - B`**，以列表/主数据为准）。
- `target_warehouse`：可选；不传则尝试默认 **`成品 - B`**。
- `items`：待发料的半成品物料编码与数量。
- `sales_order` / `custom_sales_order_no`：**必填**（回库推导）。
- 未跳过 PO 时：`supplier`、`po_items` 必填（与毛坯一致）。

## 七、前端对接说明

### 7.1 Server Action / 代理调用

- **endpoint**（相对站点根）：`/api/method/bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_semi_finished_outsourcing`
- **method**：`POST`
- **body**：Frappe 惯例为 `application/x-www-form-urlencoded` 时传 `json_data`（JSON 字符串）；若代理统一用 JSON body，以保持与现有毛坯委外 Action 一致为准。
- **Cookie / Authorization**：需已登录 ERPNext 用户会话或等价认证。

示例（概念，以项目内实际封装为准）：

```ts
const endpoint =
  "/api/method/bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_semi_finished_outsourcing";

await fetch(`${erpBaseUrl}${endpoint}`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Accept: "application/json",
    // Cookie 或 API Key 按部署约定
  },
  credentials: "include",
  body: JSON.stringify({ json_data: payload }),
});
```

### 7.2 代理白名单

若 Next.js / FastAPI 等层维护 **可调用的 method 白名单**，请 **追加** 上述完整方法路径（与 `submit_blank_outsourcing` 并列）。

### 7.3 与毛坯 Action 的关系

- 请求体字段与成功/失败解析可与 `submit_blank_outsourcing` **共用同一套 TypeScript 类型**；仅 **endpoint** 与（可选）默认展示文案不同。
- 半成品 MES 页应在接口就绪后将 `enableErpSubmit` 指向本方法。
