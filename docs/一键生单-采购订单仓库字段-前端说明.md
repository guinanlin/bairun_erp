# 一键生单 / 批量保存采购单 — 仓库字段说明（给前端联调）

本文说明：调用 `save_purchase_orders`（及单张 `save_purchase_order`）时，**采购订单明细行上的「仓库」由谁决定**，以及前端应如何传参，避免出现「所有行都落到同一仓（例如全部原料仓）」而业务上需要按行区分的情况。

---

## 1. 现象与背景

- 若界面上生成的采购订单**每一行仓库都相同**（例如全部为「原料仓」），通常是因为请求里**只设置了整张单的默认仓**，或**未按行传入仓库**，后端会统一回落到同一套默认值。
- **后端不会**根据物料主数据默认仓库、BOM 行仓库等自动推断收货仓；**以接口传入为准**。

---

## 2. 后端约定（优先级）

实现位置：`bairun_erp.utils.api.buying.purchase_order_add` 中 `_prepare_po_header` / `_prepare_po_items`。

对 **每一行** `items[]`：

| 优先级 | 字段 | 说明 |
|--------|------|------|
| 1（最高） | `items[i].warehouse` | 该行目标收货仓。有值则优先使用（经 `_resolve_warehouse` 解析，见下文）。 |
| 2 | `order_data.set_warehouse` | 整张采购单的「默认仓库」。当该行**未传** `warehouse` 时使用。 |
| 3（兜底） | 公司下任意一个非组仓库 | 当 1、2 都无法得到有效仓库时，取该公司 `Warehouse` 中 `is_group = 0` 的**第一条**记录（**仅作兜底，不保证业务含义**）。 |

主表字段 **`set_warehouse`** 与上述第 2 步一致：来自 `order_data.set_warehouse`，解析规则相同。

**结论：** 需要「按物料 / 按 BOM 行 / 按业务规则」落到不同仓库时，**必须在 `items` 每一行传入正确的 `warehouse`**（或按单拆分多张 PO 并分别设置 `set_warehouse`）。

---

## 3. `warehouse` 传什么格式

后端会对字符串做 **`_resolve_warehouse`** 解析，便于前端传简称：

1. 若已是 ERPNext 中存在的 **`Warehouse.name`**（完整链接值），直接使用。
2. 否则尝试 **`{简称} - {公司缩写}`**（如 `半成品 - B`）。
3. 否则在公司下、非组仓库中，按 **`warehouse_name` 等于传入值**，或 **`name` 包含** 传入关键字做匹配（取第一条）。

**建议：** 联调时优先与主数据/列表接口对齐，传 **与系统中 `Warehouse` 一致的 `name`**，避免模糊匹配到错误仓库。

---

## 4. 前端需要落实的事项（请联调同学自检）

1. **业务上每行应有明确收货仓时**  
   - 在组装 `order_data_list` 中每个 `order_data` 时，为 **`items` 每一行写入 `warehouse`**，值与业务规则一致（例如来自 BOM 行、物料默认仓、或界面用户选择）。  
   - **不要**仅依赖整张单的 `set_warehouse` 又期望各行不同。

2. **仅整张单同一收货仓时**  
   - 可只传 **`set_warehouse`**（主表），行上可不传 `warehouse`；此时所有行会一致使用该默认仓。

3. **批量接口**  
   - 路径：`POST /api/method/bairun_erp.utils.api.buying.purchase_order_add.save_purchase_orders`  
   - Body：`json_data.order_data_list` 为数组，**每个元素是一张 PO 的 `order_data`**，每张单独立按上表规则解析仓库。

4. **联调核对**  
   - 保存后在 ERPNext 打开采购订单，检查 **明细行「仓库」** 是否与预期一致；若仍全部相同，请抓包确认请求 JSON 里是否缺少 `items[].warehouse` 或是否被统一写成同一值。

---

## 5. 请求示例（按行区分仓库）

```json
{
  "json_data": {
    "order_data_list": [
      {
        "supplier": "某供应商",
        "company": "BR",
        "transaction_date": "2026-03-21",
        "set_warehouse": "原料仓 - BR",
        "items": [
          {
            "item_code": "RM-001",
            "qty": 100,
            "rate": 1.0,
            "warehouse": "原料仓 - BR"
          },
          {
            "item_code": "SF-002",
            "qty": 50,
            "rate": 2.0,
            "warehouse": "半成品仓 - BR"
          }
        ]
      }
    ]
  }
}
```

（`warehouse` 的具体 `name` 以当前站点主数据为准。）
