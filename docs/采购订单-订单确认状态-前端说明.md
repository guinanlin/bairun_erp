# 采购订单「订单确认状态」— 前端对接说明

## 文档信息

| 项目 | 说明 |
|------|------|
| 适用站点 | 以实际迁移后的站点为准（如 `site2.local`） |
| 面向对象 | 前端（采购订单 / 采购未交等页面） |
| 字段含义 | 整单维度的「订单确认状态」，**不是**明细行字段 |

---

## 1. 字段定义（重要）

**`order_confirmation_status` 定义在采购订单头表上，不在子表 `items` 里。**

| 项目 | 值 |
|------|-----|
| DocType | `Purchase Order`（采购订单**主表**） |
| 字段名 | `order_confirmation_status` |
| 类型 | `Data`（字符串），长度 140 |
| 标签 | 订单确认状态 |
| 自定义来源 | `bairun_erp` 模块 Custom Field：`Purchase Order-order_confirmation_status` |

业务上可与内部约定取值一致（例如：空、`订单已确认`、`毛坯已收`、`在制` 等），后端不做枚举强校验。

---

## 2. 前端读写方式

### 2.1 标准资源接口（`getDoc` / `GET /api/resource/Purchase Order/{name}`）

- 在**文档根对象**上读取、赋值：  
  `doc.order_confirmation_status`
- **不要**在 `doc.items[i]` 上找该字段（子表 `Purchase Order Item` 上**没有**这个字段）。

保存、提交时与主表其它字段一样随整单提交即可。

### 2.2 采购未交专用列表接口

若调用 `get_purchase_order_unfulfilled_list`（采购未交扁平行列表）：

- 响应里每行仍可能带有 `order_confirmation_status` 键；
- 数值来自**该采购单头表**的同一字段；同一 `purchase_order` 下的多行明细，该值**相同**（重复展示属正常）。

---

## 3. 与旧说明的差异（若文档曾写子表）

若历史需求文档写过「`Purchase Order Item.custom_order_confirmation_status`」或「在行上」等表述，**以本文为准**：当前实现为**头表字段** `Purchase Order.order_confirmation_status`。

---

## 4. 部署说明（供联调）

站点需执行迁移后，数据库表 `tabPurchase Order` 才会出现列 `order_confirmation_status`。未迁移前接口可能取不到该字段。
