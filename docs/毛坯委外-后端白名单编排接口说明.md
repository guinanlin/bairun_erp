# 毛坯 MES 委外 — 后端白名单编排接口（MR + Stock Entry + PO）

## 一、接口概览

| 项目 | 说明 |
|------|------|
| **方法名** | `submit_blank_outsourcing` |
| **完整路径** | `bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_blank_outsourcing` |
| **ERPNext URL** | `POST /api/method/bairun_erp.utils.api.mes.blank_outsourcing_submit.submit_blank_outsourcing` |
| **认证** | 需登录（`allow_guest=False`），HTTP 方法 **POST** |
| **用途** | 单入口顺序：Material Request（Material Issue，提交）→ Stock Entry（Material Issue，提交）→ Stock Entry（Material Receipt，**草稿**半成品回库）→ Purchase Order（可选跳过，提交/草稿依 `save_purchase_order`） |

通过 FastAPI / Next.js 代理时，将 `endpoint` 设为上述路径，`method` 为 POST，`json_data` 为下文参数。

---

## 二、业务与顾问确认（PO / 关联）

- **PO 行** `po_items` 的物料、单价、税目须与财务/顾问约定；本接口不代替主数据规则。
- **MR 目标仓**：库存类物料在 ERPNext 中要求填写 **Target Warehouse**。未传 `target_warehouse` 时，若存在与公司一致的标准半成品仓 **`半成品 - B`**（与 [`blank_list`](../utils/api/stock/blank_list.py) 常量一致），将自动用作 MR 行目标仓；否则须显式传入与发料仓**不同**的有效仓库。
- **Material Request 类覆盖**：`bairun_erp` 通过 `override_doctype_class` 在 `validate_from_warehouse` 前清除「目标仓=发料仓」的无效默认（`set_missing_values` 导致），避免与标准校验冲突。
- **Stock Entry 定制字段**（出库单主表，见 `bairun_erp/custom/stock_entry.json`）：有传且主数据存在时写入  
  - `custom_customer_order` ← `sales_order` / `custom_sales_order_no`（Link **Sales Order**）  
  - `custom_outsourcing_supplier` ← `supplier` / `outsourcing_supplier`（Link **Supplier**）  
  用于待委外/已委外列表按销售订单、委外供应商汇总与筛选。仅 MR+SE 时也建议传 `supplier`（或 `outsourcing_supplier`），否则 SE 上委外供应商为空。
- **新增回库步骤（Material Receipt）**：在出库 SE 之后自动创建半成品回库单，**仅保存为草稿**（`docstatus=0`），由仓管确认真实数量后再提交。  
  - **回库物料**：由 `items` 中已发**毛坯**在启用 BOM 中的**直接父项**推导（例如 BOM：成品→电镀内盖→PP 内盖毛坯，则回库为电镀内盖，而非毛坯）；再用 `sales_order` 各销售行的 BOM 树校验该父项子树确实包含所发毛坯（避免歧义）。  
  - **回库数量**：按销售订单 + **多级 BOM 展开**（`BOMTree` exploded 倍数）计算半成品应回量；若 SO 行即为该半成品则直接累计行数量（不受已发毛坯数量限制）。

## 三、请求参数（`json_data`）

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `company` | str | 否 | 默认用户默认公司 / Global Defaults |
| `from_warehouse` / `set_from_warehouse` | str | 是 | 发料仓；支持简称，内部 `_resolve_warehouse` 解析 |
| `target_warehouse` / `to_warehouse` | str | 条件 | MR 行目标仓；与发料仓须不同；未传时尝试默认 `半成品 - B` |
| `items` | list | 是 | 每行：`item_code`, `qty`，可选 `schedule_date`, `uom`, `from_warehouse`, `target_warehouse` |
| `supplier` / `outsourcing_supplier` | str | 条件 | 未 `skip_purchase_order` 时必填；仅 MR+SE 时可选，但建议填写以便 SE 写入 `custom_outsourcing_supplier` |
| `skip_purchase_order` | bool | 否 | 为 true 时仅 MR + SE |
| `po_items` | list | 条件 | 未跳过时必填；结构同 `save_purchase_order` 子表（`item_code`, `qty`, `rate` 等） |
| `purchase_order` | dict | 否 | 合并进 PO 头/税（见 `purchase_order_add`） |
| `sales_order` / `custom_sales_order_no` | str | **是** | 销售订单 name；用于回库半成品推导与数量计算；写入 MR `custom_sales_order_no`（若存在），并写入 SE `custom_customer_order`（若存在） |
| `custom_business_type` | str | 否 | MR 业务类型自定义选项 |
| `transaction_date` / `schedule_date` | str | 否 | 日期 |
| `mr_naming_series` | str | 否 | 默认 `MAT-MR-.YYYY.-` |
| `idempotency_key` | str | 否 | 成功后可重放同一结果；失败需 `force_retry=true` 再试 |
| `force_retry` | bool | 否 | 删除同键非成功日志后重试 |

## 四、响应格式

**成功：**

```json
{
  "success": true,
  "replayed": false,
  "operation_id": "…",
  "material_request_name": "MAT-MR-…",
  "stock_entry_name": "MAT-STE-…",
  "receipt_stock_entry_name": "MAT-STE-…",
  "purchase_order_name": "PUR-ORD-…"
}
```

`skip_purchase_order` 时 `purchase_order_name` 为 `null`。`replayed` 为 true 表示幂等重放。

**失败（示例）：**

```json
{
  "success": false,
  "error_code": "STOCK_ENTRY_FAILED",
  "message": "…",
  "material_request_name": "MAT-MR-…",
  "stock_entry_name": null,
  "receipt_stock_entry_name": null,
  "purchase_order_name": null,
  "operation_id": "…"
}
```

`error_code` 枚举：`VALIDATION_ERROR`, `PERMISSION_DENIED`, `MATERIAL_REQUEST_FAILED`, `STOCK_ENTRY_FAILED`, `RECEIPT_STOCK_ENTRY_FAILED`, `PURCHASE_ORDER_FAILED`, `IDEMPOTENCY_CONFLICT`。

## 五、审计与幂等

- DocType：**MES Blank Outsourcing Log**（`idempotency_key` 唯一，可为空多条）。
- 记录：`status`（Processing / Success / Partial / Failed）、三单号、耗时、`request_summary`、错误信息。
